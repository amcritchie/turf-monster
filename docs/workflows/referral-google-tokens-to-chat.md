# Workflow: Referral link ‚Üí Google signup ‚Üí buy tokens ‚Üí enter ‚Üí chat

**Trigger:** Visitor clicks a marketing link like `/landing/:slug` (or any URL with `?reference=‚Ä¶`) for a paid on-chain contest.
**Actors:** Visitor ‚Üí Google OAuth ‚Üí Rails (`LandingPagesController` / `OmniauthCallbacksController` / `TokensController` / `ContestsController` / `MessagesController`) ‚Üí Stripe ‚Üí Sidekiq (`TokenPurchaseJob`) ‚Üí Solana RPC (managed-wallet `mint_entry_token` + `enter_contest_with_token`) ‚Üí ActionCable.
**Outcome:** `users` row with `reference = "<landing-slug>"`, a `stripe_purchases` row in status `minted` with 3 on-chain `EntryTokenAccount` PDAs (one consumed), an `entries` row in status `complete` with `onchain_tx_signature`, a `messages` row broadcast on `[contest, :messages]`.
**Preconditions:** Contest is `open`, `onchain`, has `chat_enabled = true` (default ‚Äî `db/schema.rb:66`), and is backed by a configured `SeasonConfig.current_season_id` (`contests_controller.rb:296`). Stripe enabled. Visitor not in a blocked geo state.

## Sequence

1. **Visit landing page (first touch)** ‚Äî `app/controllers/landing_pages_controller.rb:18`
   - `cookies[:reference] = { value: @landing_page.slug, expires: 30.days }` (only if blank ‚Äî an explicit `?reference=` already captured by `ApplicationController#capture_reference` at `app/controllers/application_controller.rb:51-56` wins). Cookie lives 30 days across the funnel.

3. **Guest selects 6 picks, attempts to enter** ‚Äî `app/views/contests/_turf_totals_board.html.erb:899` (`confirmEntry()`). Picks live in Alpine state (`selections` / `selectionOrder`); no server cart is persisted yet for guests. Lines 901-904: `Alpine.store('session').isGuest === true` ‚Üí calls `showLoginModal()`.
4. **Auth modal opens at `credentials` step** ‚Äî `app/views/contests/_turf_totals_board.html.erb:390-402` opens `Alpine.store('modals').open('auth', { step: 'credentials', mode: 'signup', ‚Ä¶ })`. Wizard partial: `app/views/modals/_auth.html.erb`. If blocked, falls back to full-page form POST `/auth/google_oauth2` at line 649-661 (cart still saved).
6. **Google round-trip + OmniAuth callback** ‚Äî `app/controllers/omniauth_callbacks_controller.rb:22` (`#create`). 8. **In-modal token pack picker ‚Üí Stripe Checkout** ‚Äî `app/controllers/tokens_controller.rb:11` (`#stripe_checkout`). User picks `trio` (3 tokens for $4900 ‚Äî `app/models/stripe_purchase.rb:14-15`). 9. **Stripe payment ‚Üí webhook** ‚Äî `app/controllers/webhooks/stripe_controller.rb:8` (`#create`, CSRF + auth skipped on lines 3-6). `Stripe::Webhook.construct_event` verifies signature (line 14). `event.type == "checkout.session.completed"` routes to `handle_checkout_completed` (line 66): `StripeCheckoutValidator` re-fetches the session and asserts `payment_status` / `livemode` / `kind` / `amount` (line 75 ‚Äî OPSEC-008/032/033). 10. **On-chain token mint (the "minted" step is real on-chain, not DB)** ‚Äî `app/jobs/token_purchase_job.rb:16`. Each mint creates an on-chain `EntryTokenAccount` PDA owned by the user's managed wallet. Signatures persist incrementally to `stripe_purchases.mint_tx_signatures` after every mint (line 75) so a mid-loop crash resumes at `already_minted = signatures.length` (line 57-59). 12. **Hold to Confirm fires entry submission** ‚Äî `window` event `hold-confirm-entry` ‚Üí `confirmEntry()` at `app/views/contests/_turf_totals_board.html.erb:899`. Web2 (managed-wallet) branch: POST `/contests/:slug/enter` (line 1040). The cart picks were replayed to the server via `replaySelectionsToServer()` (line 666-...) right after signup (line 475 ‚Äî survives the Stripe detour because they're cart rows in Postgres at that point). 13. **Redirect lands on contest page** ‚Äî `contests_controller.rb` returns `redirect: contest_path(@contest)` in JSON. `tokens-submitted` step renders `_entry_confirmed` card (`app/views/modals/auth/_tokens.html.erb:110-116`) with the CTA. ## Failure modes

- **Reference cookie collision** ‚Äî landing page only sets if `cookies[:reference].blank?` (`landing_pages_controller.rb:18`), and `ApplicationController#capture_reference` likewise no-ops if present (`application_controller.rb:53`). First touch wins; later landing visits leave the cookie alone. ‚Üí silent; correct behaviour.
- **Google email already linked to unverified existing user** ‚Äî `User.from_omniauth` returns `:requires_verification` (`user.rb:62-63`). Controller surfaces "verify your existing account first" alert (`omniauth_callbacks_controller.rb:90-91`). No silent merge (OPSEC-005). Reference stays in the cookie for the eventual signup.
- **Profile not complete** ‚Äî N/A on this app: `ensure_username` auto-fills the username via `Studio::UsernameGenerator.generate` at `user.rb:312-314`, so `profile_complete?` is always true post-create.
- **Stripe webhook signature mismatch / replay** ‚Äî `Webhooks::StripeController#create` returns 400 on bad signature (line 19-20) and 200 + log on test-event-in-production (line 29-32). Validator rejects mismatched payment_status/livemode/kind/amount with logged `validator_rejected` (line 77-80).
- **Partial mint failure (1 of 3 tokens minted, RPC crashes)** ‚Äî `TokenPurchaseJob` persists each signature before continuing (`token_purchase_job.rb:75`). On Sidekiq retry, loop resumes at `already_minted = signatures.length` (line 57-59). `mark_failed_unless_minted!` (line 99) never downgrades a `minted` row.
- **User refreshes during the Stripe round-trip** ‚Äî cart picks already POSTed to the server as cart-status `entries` + `selections` rows by `replaySelectionsToServer` (`_turf_totals_board.html.erb:475`). The processing-tab `BroadcastChannel` won't reach a closed contest tab, but the next contest visit reads cart rows back from the DB. On-chain mint completes regardless (job is async).
- **`/tokens/status` polling times out (>45s)** ‚Äî modal flips to `tokens-error` (`_turf_totals_board.html.erb:761-768`) with "Refresh in a moment" copy.
- **`ContestsController#enter` race ‚Äî contest full mid-flight** ‚Äî `with_lock` at `contests_controller.rb:288` + `active_count >= @contest.max_entries` check at line 290 raises "Contest is full"; the response 422s and the modal surfaces the error.
- **Chat message rate-limit hit** ‚Äî `MessagesController#posting_too_fast?` returns 429 (`messages_controller.rb:14`). User sees "You're posting too fast" in the composer.
- **ActionCable broadcast failure** ‚Äî `Message#broadcast_new_message` wraps in `rescue` and logs to `ErrorLog` (`message.rb:50-51`) so the controller request still returns `{ ok: true }`. Other subscribers miss the message; the poster sees it (Turbo Drive renders locally via the controller response ‚Äî actually no, this isn't ideal; the local sender also waits on the broadcast). ‚Üí Inspect `error_logs` for `Message` targets.
- **Chargeback / refund post-mint** ‚Äî `handle_dispute` / `handle_refund` in `webhooks/stripe_controller.rb:111-136` flag the user (`payment_risk_flag`) and freeze them (`freeze_for_payment_risk!`). Future buys + entries blocked; operator unfreezes via console.
